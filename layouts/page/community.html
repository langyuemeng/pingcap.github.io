How to become a TiDB Committer?

First of all, you need to get involved and be a Contributor. A new Committer must be nominated by any existing Maintainer or PMC member, and is generated through the voting of PMC members. A Committer must have accomplished one or more of the following items:

  • Your commits have been approved and merged by Maintainers for over ten times.
  • You have implemented a complete feature.
  • You have fixed a vital bug.

If a PR submitted to the TiDB/TiKV/ TiSpark/PD/ TiDB Operator/Parser/ Docs/Docs-cn projects by you is approved and merged, then you become a TiDB Contributor.

+
Your First Contribution

Your journey of becoming a contributor and committer starts from here: improving docs, improving code, giving talks, organizing meetups, etc. You can get your unique TiDB mug designed only for TiDB contributors. Start contributing now!

  • Fork the project to your GitHub account
  • Git Clone to your local repository
  • Create a new branch based on the master branch
  • Code and test in your local repository
  • Commit and push your commit to the remote repository
  • File a PR
  • Code review: After we receive your PR, we will review your code, so please keep an eye on the conversations and comments of your PR. If you receive two LGTMs, your PR is approved and is to be merged! Congratulations on being a new TiDB contributor! Contact community@pingcap.com for your special swags as contributors.
+
New to the Community?

If you have never been working on any open source projects, you can still contribute to the community. You can

  • Try TiDB! If you like it, star, share, and recommend it.
  • Look for GitHub issues with the 'for new contributors' or 'help wanted' labels. These lables are ususally easy and fun to start with.
  • Dig deeper, read a design doc, e.g. How we build TiDB.
  • Start with improving the documentation. You can search open issues to find inspiration for what to work on.
  • Report bugs by creating issues or fix known issues
  • There's always code that can be clarified and variables or functions that can be renamed or commented.
  • Share your story with TiDB by creating case studies, technical blogs or give a talk at a local meetup or conferences.
